## 项目主要需求

### 一、项目概述

LegacyTree 是一个数字遗产与家谱管理平台，帮助用户构建可视化家族树、记录家族故事、保存老照片并规划数字遗产。本项目为纯前端应用，所有数据通过 localStorage 和 IndexedDB 进行本地持久化，敏感数据使用 Web Crypto API 进行加密存储。家谱树使用 D3.js 渲染交互式可视化，照片修复使用 Canvas API 实现基础滤镜效果。

### 二、核心数据模型

#### 1. 家族成员 FamilyMember

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| name | string | 姓名 |
| gender | Gender | 性别（MALE/FEMALE/OTHER） |
| birthDate | string | 出生日期（YYYY-MM-DD，可空表示未知） |
| deathDate | string | 逝世日期（可空） |
| birthPlace | string | 出生地 |
| bio | string | 个人简介 |
| avatar | string | 头像 URL/base64 |
| parentId | string | 父/母节点 ID（用于构建树） |
| spouseIds | string[] | 配偶 ID 列表（支持多段婚姻） |
| childrenIds | string[] | 子女 ID 列表 |
| generation | number | 世代编号（第一代为 1） |

**树形结构说明**：通过 parentId 和 childrenIds 构建家谱树，配偶关系通过 spouseIds 关联。D3.js 树布局需要同时展示血缘关系和婚姻关系。

**业务动作**：添加成员 → 编辑信息 → 关联配偶/子女 → 查看成员详情 → 删除成员（级联处理子节点）

#### 2. 家族故事 Story

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| authorId | string | 撰写者（家庭成员 ID 或用户） |
| memberId | string | 关联的家族成员 ID |
| title | string | 故事标题 |
| content | string | 故事正文（支持 Markdown） |
| mediaUrls | string[] | 附带的图片/视频 URL 列表 |
| category | StoryCategory | 分类（回忆/成就/趣事/家训） |
| date | string | 故事发生日期 |
| isPublic | boolean | 是否公开 |

**业务动作**：撰写故事 → 编辑修改 → 按分类组织 → 时间线浏览 → 分享导出

#### 3. 老照片 Photo

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| uploaderId | string | 上传者 ID |
| memberId | string | 关联的家族成员 ID |
| imageUrl | string | 原始图片 URL/base64 |
| caption | string | 照片说明 |
| year | number | 拍摄年份（大致年份） |
| location | string | 拍摄地点 |
| people | string[] | 照片中出现的人物 ID 列表 |
| isRestored | boolean | 是否已修复 |
| restoredUrl | string | 修复后的图片 URL/base64 |

**业务动作**：上传照片 → 标注信息（年份、地点、人物） → AI 修复（Canvas API 滤镜） → 收藏/取消收藏 → 按时间轴浏览

#### 4. 遗产规划 LegacyPlan

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| memberId | string | 关联成员 ID |
| type | LegacyType | 类型（遗嘱意向/数字资产/纪念品/信件） |
| content | string | 规划内容 |
| beneficiaries | string[] | 受益人成员 ID 列表 |
| status | 'draft' \| 'finalized' \| 'archived' | 状态 |
| createdAt | string | 创建时间 |

**业务动作**：创建规划 → 编辑内容 → 指定受益人 → 定稿归档

### 四、核心页面

#### 1. `/tree` 家谱树

- **D3.js 交互式家族树**：缩放（zoom）、平移（pan）、点击节点展开/折叠子树
- **节点展示**：头像 + 姓名 + 生卒年份，已故成员特殊边框样式
- **配偶连线**：婚姻关系用虚线连接
- **操作**：点击节点弹出详情面板，右键菜单（编辑、添加子女、添加配偶、删除）
- **工具栏**：缩放控制、全屏模式、搜索成员、居中视图、导出图片
- **新增成员**：点击空白区域或使用工具栏按钮
- 组件：使用 `FamilyTreeView`、`TreeNode`、`TreeControls`

#### 2. `/members/:id` 成员详情

- **个人信息卡**：头像（使用 `MemberAvatar` 组件）、姓名、性别、生卒日期、出生地、简介
- **家族故事**：该成员关联的故事列表，时间线展示（使用 `TimelineView`）
- **老照片**：该成员关联的照片网格（使用 `MediaGallery`）
- **关联成员**：配偶、子女、父母的快捷导航链接
- **遗产规划**：与该成员相关的遗产规划列表
- **操作**：编辑信息、撰写新故事、上传照片、创建遗产规划

#### 3. `/stories` 家族故事

- **故事列表**：卡片展示，每张卡片含标题、摘要、分类标签（`StoryCategory`）、关联成员、日期
- **分类筛选**：按回忆/成就/趣事/家训筛选
- **时间线视图**：切换到时间线模式（使用 `TimelineView`），按时间轴排列
- **故事详情**：展开阅读全文，查看附带媒体（使用 `MediaGallery`）
- **撰写故事**：富文本编辑器，选择关联成员，选择分类，上传媒体

#### 4. `/photos` 老照片馆

- **照片墙**：瀑布流布局展示所有照片缩略图
- **时间轴视图**：按年份组织照片（使用 `TimelineView`）
- **照片详情**：点击查看大图，展示标注信息（年份、地点、人物）
- **AI 修复**：点击"修复"按钮，使用 Canvas API 应用滤镜（去噪、增强对比度、锐化），展示修复前后对比
- **标注操作**：编辑照片说明、年份、地点，标注照片中的人物
- **组件**：使用 `MediaGallery` 展示照片集合

#### 5. `/legacy` 遗产规划

- **规划列表**：表格展示所有遗产规划，展示类型图标、关联成员、状态、创建时间
- **创建规划**：选择类型（`LegacyType`），关联成员，编写内容，指定受益人
- **编辑/定稿**：修改规划内容，状态流转（草稿 → 定稿 → 归档）
- **提醒功能**：定期提醒查看和更新规划

#### 6. `/settings` 设置

- **加密设置**：设置/修改数据加密密码
- **数据管理**：导入/导出数据，清除数据
- **主题设置**：深色/浅色模式切换

### 五、横切关注点

#### 1. 数据隐私与加密

**实现方式**：使用 Web Crypto API (AES-GCM) 对敏感数据进行加密后存储到 IndexedDB

**触达层面**：
- **DB 层**：`src/db/` 模块在写入前自动加密，读取后自动解密
- **加密工具**：`src/utils/crypto.ts` 封装 Web Crypto API 的 encrypt/decrypt 函数
- **导出功能**：导出的 JSON 文件中的敏感字段为加密状态，导入时需提供密码解密

**加密策略**：
- 用户首次使用时设置加密密码，密码通过 PBKDF2 派生密钥
- 密钥仅在内存中保存（sessionStorage），关闭页面后需重新输入密码
- 敏感字段包括：成员个人信息、故事内容、遗产规划内容
- 照片数据（base64）不做加密（体积过大），但标注信息加密

#### 2. 数据导入导出

**支持格式**：
- **JSON**：完整的家谱数据导出/导入（包含所有实体和关系）
- **GEDCOM**：标准家谱格式兼容导入（解析 .ged 文件，转换为内部数据模型）

**触达层面**：
- **全部 Store**：导出时遍历所有 store 收集数据，导入时写入所有 store
- **文件处理工具**：`src/utils/export.ts` 处理导出逻辑，`src/utils/gedcom-parser.ts` 解析 GEDCOM 格式
- **加密集成**：导出时可选择是否加密，导入时需要密码（如果数据已加密）

### 六、技术实现要求

- **状态管理**：Pinia，每个实体一个独立 store 文件
- **IndexedDB**：封装为统一的 `src/db/` 模块，在 IndexedDB 操作外层包裹加密/解密逻辑
- **D3.js 家谱树**：使用 `d3-hierarchy` 布局，自定义节点渲染，支持缩放/平移（`d3-zoom`）
- **Canvas 照片修复**：`Canvas API` 实现图像滤镜（亮度/对比度/锐化/去噪），提供修复前后对比视图
- **路由**：Vue Router 4，路由配置独立文件
- **UI 组件库**：Naive UI，提供表格、表单、抽屉、对话框等基础组件
- **类型安全**：所有数据模型必须有 TypeScript 类型定义，放在 `src/types/` 目录

### 七、全局异常处理

**前端统一错误处理**：
- 封装 IndexedDB 操作的统一 try-catch 层，操作失败时使用 Naive UI 的 `n-message` 组件展示友好错误提示
- 加密/解密失败的特殊处理：提示用户检查密码是否正确
- D3.js 渲染异常的兜底处理：家谱树渲染失败时展示降级的列表视图
- 所有异步操作（文件导入导出、照片处理）必须有 loading 状态和错误提示

## 文件结构强制清单

**严禁将多个职责合并到单一文件中。** 每个文件必须只承担一个明确的职责。例如：不能在一个组件中同时定义数据获取逻辑和渲染逻辑（应通过 hooks/Pinia action 分离）；不能在一个 store 中管理多个不相关的实体状态；不能在 DB 层文件中混入加密逻辑（加密应通过独立的 crypto 工具模块）。

```
src/
├── stores/        # familyStore.ts, storyStore.ts, photoStore.ts, legacyStore.ts, settingsStore.ts
├── types/         # family.d.ts, story.d.ts, photo.d.ts, legacy.d.ts
├── constants/     # enums.ts, default-templates.ts
├── components/common/  # MemberAvatar, TimelineView, MediaGallery, EmptyState, ConfirmDialog
├── components/tree/    # FamilyTreeView, TreeNode, TreeControls
├── hooks/         # useFamily(), useStory(), usePhoto(), useEncryption()
├── pages/         # FamilyTree, MemberDetail, Stories, Photos, Legacy, Settings
├── router/        # index.ts, routes.ts, guards.ts
├── db/            # IndexedDB 封装（family-db.ts, story-db.ts, photo-db.ts, legacy-db.ts, index.ts）
├── utils/         # gedcom-parser.ts, crypto.ts, export.ts, image-filter.ts
└── assets/        # 默认头像、空状态插画、图标
```
